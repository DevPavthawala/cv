Q1 Prove: \(\alpha \cdot h*(f+g) = h*(\alpha f) + h*(\alpha g)\)
Proof

Start from the convolution definition:

$$ (h*(f+g))(x)=\int h(\tau)\bigl(f(x-\tau)+g(x-\tau)\bigr)\,d\tau $$

Multiply both sides by scalar \(\alpha\):

$$ \alpha\,(h*(f+g))(x)=\int h(\tau)\bigl(\alpha f(x-\tau)+\alpha g(x-\tau)\bigr)\,d\tau $$

Split the integral (linearity of integration):

$$ =\int h(\tau)\,\alpha f(x-\tau)\,d\tau + \int h(\tau)\,\alpha g(x-\tau)\,d\tau = (h*\alpha f)(x)+(h*\alpha g)(x) $$

Hence proved.

Q2 Prove: convolving twice with \(G_\sigma\) equals one convolution with \(G_{\sigma\sqrt{2}}\)
Proof

Key property: the convolution of two Gaussians is a Gaussian whose variance is the sum of the individual variances:

$$ \mathcal{N}(0,\sigma_1^2)*\mathcal{N}(0,\sigma_2^2)=\mathcal{N}(0,\sigma_1^2+\sigma_2^2) $$

Setting \(\sigma_1=\sigma_2=\sigma\):

$$ G_\sigma * G_\sigma = G_{\sqrt{\sigma^2+\sigma^2}} = G_{\sigma\sqrt{2}} $$

Hence proved.

Q3 Prove: any linear shift-invariant (LSI) operator \(T\) can be represented as convolution
Proof

Step 1 — Define impulse response. Let \(\delta\) be the unit impulse. Define kernel:

$$h = T\{\delta\}$$

Step 2 — Decompose signal. Any discrete signal can be written as a weighted sum of shifted impulses:

$$f = \sum_k f[k]\,\delta(\cdot - k)$$

Step 3 — Apply linearity.

$$T\{f\} = \sum_k f[k]\,T\{\delta(\cdot-k)\}$$

Step 4 — Apply shift-invariance. Since \(T\) is shift-invariant, \(T\{\delta(\cdot-k)\} = h(\cdot-k)\):

$$T\{f\} = \sum_k f[k]\,h(\cdot-k) = f * h$$

Hence proved. Every LSI operator is a convolution with its impulse response.

Q4 Kernel separability and time complexity
Definition

A \(k\times k\) kernel \(K\) is separable if it can be written as an outer product of two 1D vectors:

$$K = \mathbf{a}\,\mathbf{b}^T, \quad \mathbf{a},\mathbf{b}\in\mathbb{R}^k \quad (\text{rank-1 matrix})$$

Time complexity for an \(n\times n\) image with a \(k\times k\) kernel:

Kernel typeComplexityReason
General (non-separable)\(\mathcal{O}(n^2 k^2)\)Full 2D sliding window
Separable\(\mathcal{O}(n^2 k)\)Two 1D passes (horizontal + vertical)

Separable filtering gives a speedup factor of \(k\) — significant for large kernels (e.g., \(k=15\) means 15× faster).

Q5 Kalman filter — prediction and update equations

State and measurement models:

$$x_k = F_k x_{k-1} + B_k u_k + w_k, \quad w_k\sim\mathcal{N}(0,Q_k)$$ $$z_k = H_k x_k + v_k, \quad v_k\sim\mathcal{N}(0,R_k)$$

Prediction step:

$$\hat{x}_{k|k-1} = F_k\hat{x}_{k-1|k-1} + B_k u_k$$ $$P_{k|k-1} = F_k P_{k-1|k-1} F_k^T + Q_k$$

Update (correction) step:

$$K_k = P_{k|k-1}H_k^T\bigl(H_k P_{k|k-1}H_k^T + R_k\bigr)^{-1} \quad \text{(Kalman gain)}$$ $$\hat{x}_{k|k} = \hat{x}_{k|k-1} + K_k\bigl(z_k - H_k\hat{x}_{k|k-1}\bigr)$$ $$P_{k|k} = (I - K_k H_k)\,P_{k|k-1}$$
Q6 Two main assumptions of the standard Kalman filter
  1. Linearity: Both the state transition model (\(F_k\)) and the measurement model (\(H_k\)) must be linear.
  2. Gaussian noise: Process noise \(w_k\) and measurement noise \(v_k\) are zero-mean, white, mutually independent Gaussian distributions.
Q7 Three other filters beyond the standard Kalman filter
  1. Extended Kalman Filter (EKF): Linearizes nonlinear models via first-order Taylor expansion (Jacobians). Advantage: handles mild nonlinearity with low computational cost.
  2. Unscented Kalman Filter (UKF): Propagates a set of deterministically chosen sigma points through the nonlinear function to estimate mean and covariance. Advantage: more accurate than EKF for stronger nonlinearities, no Jacobian required.
  3. Particle Filter: Represents the posterior distribution using a set of weighted random samples (particles). Advantage: handles non-Gaussian, multimodal, and highly nonlinear distributions.
Q8 Hough transform for circles — unknown radius, known gradient direction \(\theta\)
Pseudocode
// Input: edge map + gradient direction θ per pixel // Output: detected circles (a, b, r) Initialize accumulator A(a, b, r) = 0 for all (a,b) in image space, r ∈ [r_min, r_max] For each edge pixel (x, y) with gradient direction θ(x,y): For each radius r ∈ [r_min, r_max]: // Vote for center along gradient direction a₁ = x − r·cos(θ) b₁ = y − r·sin(θ) A(a₁, b₁, r) += 1 // Also vote in opposite direction a₂ = x + r·cos(θ) b₂ = y + r·sin(θ) A(a₂, b₂, r) += 1 Find local maxima in A(a, b, r) above threshold τ Return (a, b, r) for each peak
The gradient direction constrains the center to lie along one of two rays, reducing a 2D locus (full circle in standard Hough) to just two candidate points per radius — significantly fewer votes and fewer false positives.
Q9 Epipolar constraint for the uncalibrated case

Two points \(x\) and \(x'\) (in homogeneous image coordinates) are correspondences if and only if:

$$x'^T F x = 0$$

where \(F\) is the fundamental matrix (a \(3\times3\) rank-2 matrix). This encodes the geometric relationship between two uncalibrated cameras — for a given point \(x\) in image 1, its corresponding point \(x'\) in image 2 must lie on the epipolar line \(\ell' = Fx\).

Q10 PCA shape model: 28 shapes, 54 2D points, 8 parameters
Step-by-step

1. Vectorize each shape. Each shape has 54 × 2 = 108 coordinates:

$$s_i \in \mathbb{R}^{108}, \quad i = 1,\ldots,28$$

2. Form data matrix.

$$S = [s_1, \ldots, s_{28}] \in \mathbb{R}^{108 \times 28}$$

3. Compute mean and centre.

$$\bar{s} = \frac{1}{28}\sum_{i=1}^{28} s_i \in \mathbb{R}^{108}, \qquad \tilde{S} = S - \bar{s}\mathbf{1}^T \in \mathbb{R}^{108\times 28}$$

4. Compute covariance and eigen-decompose.

$$C = \frac{1}{27}\tilde{S}\tilde{S}^T \in \mathbb{R}^{108\times 108}$$

5. Keep top 8 eigenvectors.

$$P = [u_1, \ldots, u_8] \in \mathbb{R}^{108 \times 8}$$

6. Shape model (only 8 shape parameters \(b\)):

$$s \approx \bar{s} + P\,b, \quad b \in \mathbb{R}^8$$
Note: Procrustes alignment was already done, so we skip translation/rotation/scale removal here.
Q11 Nonrigid structure from motion using a shape basis (Bregler et al. 2000)

Model: The 3D shape at each frame \(t\) is a linear combination of \(K\) basis shapes:

$$S_t = \sum_{i=1}^K c_{ti}\,B_i, \quad B_i \in \mathbb{R}^{3\times P}$$

Under orthographic camera:

$$W_t = R_t S_t + T_t$$

Stacking all frames, the measurement matrix \(W \in \mathbb{R}^{2F \times P}\) has rank \(\le 3K\).

Pseudocode
Input: Tracked 2D point correspondences W (2F × P) Output: Basis shapes B_i, coefficients c_ti, rotations R_t 1. Centre tracks (remove translation T_t per frame) 2. Factorize: W ≈ M·S using truncated SVD with rank 3K 3. Enforce metric upgrade constraints on M (each 2×3 block must be a valid scaled rotation) 4. Recover R_t and c_ti from M 5. Recover basis shapes B_i from S 6. (Optional) Refine via alternating optimisation / bundle adjustment
Q12 Condition for polynomial-time multilabel MRF optimization

The pairwise (smoothness) term \(V_{pq}(L_p, L_q)\) must be submodular — equivalently, it must satisfy the metric (or semi-metric) condition:

$$V_{pq}(a,b) + V_{pq}(c,d) \le V_{pq}(a,d) + V_{pq}(c,b)$$

Under this condition, graph-cut based algorithms such as alpha-expansion and alpha-beta swap are applicable and converge to a locally optimal solution in polynomial time per move.

Common valid choices include the truncated linear and truncated quadratic penalties. Non-submodular terms make the problem NP-hard in general.

Q13 Computing the fundamental matrix \(F\) from 8 correspondences

For each correspondence \((u_i,v_i) \leftrightarrow (u'_i,v'_i)\), the epipolar constraint \(x'^T F x = 0\) expands to:

$$[u'_i u_i,\; u'_i v_i,\; u'_i,\; v'_i u_i,\; v'_i v_i,\; v'_i,\; u_i,\; v_i,\; 1]\,\mathbf{f} = 0$$

where \(\mathbf{f} = \text{vec}(F)\in\mathbb{R}^9\). Stack 8 such equations into \(A\mathbf{f} = 0\), \(A\in\mathbb{R}^{8\times9}\).

Solve: Take the right singular vector of \(A\) corresponding to the smallest singular value (via SVD).

Enforce rank-2: Decompose \(F = U\Sigma V^T\), set the smallest singular value of \(\Sigma\) to zero, and reconstruct \(F\).

Denormalize if normalized point coordinates were used.

Q14 Normalized eight-point algorithm
Input: 8+ correspondences (x_i, x'_i) 1. Normalize points in each image independently: - Translate centroid to origin - Scale so mean distance from origin = √2 - Store transforms T (image 1) and T' (image 2) 2. Build matrix A from normalized correspondences (one row per correspondence, as shown in Q13) 3. Solve A·f = 0 via SVD → reshape to F̂ (3×3) 4. Enforce rank-2 on F̂: F̂ = U diag(σ₁,σ₂,0) Vᵀ 5. Denormalize: F = T'ᵀ F̂ T 6. (Optional) normalize scale so ||F|| = 1
Why normalize? Point coordinates in pixel space have very different scales, causing poor numerical conditioning. Normalization makes the system well-conditioned and greatly improves accuracy.
Q15 Linear system for estimating camera matrix \(P\) from 3D–2D correspondences (DLT)

For 3D point \(X_i = [X_i, Y_i, Z_i, 1]^T\) with 2D projection \(x_i = (u_i, v_i)\), the relation \(x_i \sim P X_i\) yields two linear equations by eliminating the scale:

$$[-X_i^T,\; \mathbf{0}^T,\; u_i X_i^T]\,\mathbf{p} = 0$$ $$[\mathbf{0}^T,\; -X_i^T,\; v_i X_i^T]\,\mathbf{p} = 0$$

where \(\mathbf{p} = \text{vec}(P)\in\mathbb{R}^{12}\). Stacking over all \(n\) points:

$$A\,\mathbf{p} = 0, \quad A \in \mathbb{R}^{2n\times 12}$$

Solve via SVD. Minimum 6 points required (each contributing 2 equations for 11 DoF).

Q16 Camera parameters in matrix form (no distortion)
$$P = K[R \mid t]$$

Intrinsic matrix \(K\) (5 DoF):

$$K = \begin{bmatrix} f_x & s & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$$

where \(f_x, f_y\) are focal lengths in pixels, \((c_x, c_y)\) is the principal point, and \(s\) is the skew (usually 0 for modern cameras).

Extrinsic parameters: rotation \(R \in SO(3)\) (3 DoF) and translation \(t \in \mathbb{R}^3\) (3 DoF) — encode the camera pose in world coordinates.

Q17 Discrete form of \(\partial_t\phi = w|\nabla\phi|\operatorname{div}\!\left(\frac{\nabla\phi}{|\nabla\phi|}\right) + \nabla w^T\nabla\phi\)

This is the geodesic active contour / weighted level set evolution equation. Using explicit (forward Euler) time stepping with step \(\Delta t\):

$$\phi^{n+1}_{i,j} = \phi^n_{i,j} + \Delta t \left[w_{i,j}\,|\nabla\phi^n|_{i,j}\,\kappa^n_{i,j} + \nabla w_{i,j}\cdot\nabla\phi^n_{i,j}\right]$$

where the mean curvature term is:

$$\kappa^n_{i,j} = \operatorname{div}\!\left(\frac{\nabla\phi^n}{|\nabla\phi^n|+\varepsilon}\right)_{i,j}$$

All spatial derivatives (gradient \(\nabla\), divergence \(\operatorname{div}\)) are computed via central finite differences. The small \(\varepsilon\) avoids division by zero.

Context: \(\phi\) is the level set function, \(w\) is an edge-stopping weight (e.g., \(w = 1/(1+|\nabla I|^2)\)), and the zero level set of \(\phi\) is the evolving contour.
Q18 2D Distance Transform — forward/backward dynamic programming passes
Algorithm (Chamfer approximation)

Initialization: Set edge pixels to 0, all others to \(\infty\).

Forward pass (top-left → bottom-right): For each pixel update from its already-processed neighbours (e.g., left, up, up-left, up-right) by adding the corresponding chamfer weight.

Backward pass (bottom-right → top-left): For each pixel update from its remaining neighbours (right, down, down-left, down-right).

Result: Each cell contains the approximate shortest distance to the nearest edge pixel.

In the exam, three 5×5 grids are provided to show: (1) initialisation, (2) after forward pass, (3) after backward pass. Fill them in using the chamfer mask weights (e.g., 1 for horizontal/vertical, \(\sqrt{2}\approx1.414\) or 1.5 for diagonal, depending on the mask).
Part B — WS 2023/24  (21 Exercises)
Exam topics: SfM, Projections, Stereo, Image Processing, Filtering, Hough, Spectral Clustering
Q1 Similarity ambiguity matrix in SfM — how many DoF?

After factorization, the reconstruction is only defined up to a global similarity transform in 3D (scale + rotation + translation). Its homogeneous form is:

$$H_s = \begin{bmatrix} sR & t \\ \mathbf{0}^T & 1 \end{bmatrix} \in \mathbb{R}^{4\times4}$$

where \(s\in\mathbb{R}^+\), \(R\in SO(3)\), \(t\in\mathbb{R}^3\).

Degrees of freedom: 1 (scale) + 3 (rotation) + 3 (translation) = 7 DoF.

Q2 Orthographic projection matrix

Orthographic projection discards depth — only the first two rows of the rotation are used:

$$\begin{bmatrix} u \\ v \end{bmatrix} = \begin{bmatrix} r_1^T \\ r_2^T \end{bmatrix} X + \begin{bmatrix} t_x \\ t_y \end{bmatrix}$$

In homogeneous form (\(2\times4\) matrix):

$$P_{\text{ortho}} = \begin{bmatrix} r_{11} & r_{12} & r_{13} & t_x \\ r_{21} & r_{22} & r_{23} & t_y \end{bmatrix}$$
Q3 Knowns and unknowns in projective SfM

Known: 2D image point tracks \(x_{ij}\) — the observed projections of point \(j\) in view \(i\).

Unknown:

  1. Projective camera matrices \(P_i \in \mathbb{P}^{3\times4}\), one per view.
  2. Projective 3D points \(X_j \in \mathbb{P}^3\), one per scene point.
  3. A global projective gauge: the absolute frame is only defined up to a \(4\times4\) projective transformation (15 DoF ambiguity).
Q4 Minimum views for projective SfM with 13 points; general equation

Parameter count:

  • Each projective camera: 11 DoF → \(11m\) total
  • Each projective 3D point: 3 DoF → \(3n\) total
  • Global projective gauge: −15 DoF

General constraint (\(m\) views, \(n\) points):

$$2mn \ge 11m + 3n - 15$$

For \(n = 13\) points:

$$26m \ge 11m + 39 - 15 = 11m + 24 \;\Rightarrow\; 15m \ge 24 \;\Rightarrow\; m \ge 2$$

So at least 2 views are needed (by point-counting; in practice more are required for stable solutions).

Q5 Extrinsic parameters of the second camera from \(F\)

Given intrinsics \(K\) (camera 1) and \(K'\) (camera 2), recover the essential matrix:

$$E = K'^T F K$$

Decompose via SVD: \(E = U\,\text{diag}(1,1,0)\,V^T\) (enforce the correct singular value structure).

Define the helper matrix \(W = \begin{bmatrix}0&-1&0\\1&0&0\\0&0&1\end{bmatrix}\). There are four candidate solutions:

$$[R\mid t] \in \{UWV^T\mid +U_3,\; UWV^T\mid -U_3,\; UW^TV^T\mid +U_3,\; UW^TV^T\mid -U_3\}$$

where \(U_3\) is the third column of \(U\). Select the correct solution using the cheirality check: triangulate a point and verify it lies in front of both cameras (\(Z > 0\)).

Q6 Cross product \(a \times b\) as matrix multiplication
$$a \times b = [a]_\times\, b, \qquad [a]_\times = \begin{bmatrix} 0 & -a_3 & a_2 \\ a_3 & 0 & -a_1 \\ -a_2 & a_1 & 0 \end{bmatrix}$$

The skew-symmetric matrix \([a]_\times\) is also used to form the essential matrix: \(E = [t]_\times R\).

Q7 Main steps of sequential projective SfM
  1. Initialize from two views: Estimate \(F\), set \(P_1 = [I\mid\mathbf{0}]\), derive \(P_2\) from \(F\), triangulate initial point cloud.
  2. Add new view: Use 2D–3D correspondences between the existing point cloud and the new image to estimate \(P_k\) via projective resection (PnP-style).
  3. Triangulate new points: Add points visible in the new view and at least one previous view.
  4. Bundle adjustment: Jointly refine all cameras \(\{P_i\}\) and 3D points \(\{X_j\}\) by minimizing total reprojection error.
  5. Repeat steps 2–4 until all views are integrated.
Q8 Duality between shape and trajectory factorization

Both approaches factorize the same measurement matrix \(W \in \mathbb{R}^{2F\times P}\), but organize the basis differently:

Basis spans3D shape spaceTrajectory / time space

Q9 How are basis shapes estimated in trajectory factorization?

The temporal basis functions (e.g., DCT basis, low-order polynomials, or splines) are typically predefined a priori — they are not estimated from data. The trajectory coefficients (how each point's motion projects onto each basis function) are then estimated from the data via least squares or alternating minimization.

Q10 Assumptions for trajectory factorization
  1. Low-dimensional motion subspace: All point trajectories lie in a low-dimensional linear subspace (spanned by \(K\) temporal basis functions).
  2. Orthographic/weak perspective projection: The camera model is simplified so that the measurement matrix has bounded rank and the factorization is tractable.
Q11 Disparity from rectified stereo + depth equation

After rectification, epipolar lines are horizontal, so correspondence search is restricted to the same row. Disparity is:

$$d(u,v) = u_L - u_R$$

For each pixel, the match is found by minimizing a cost (SAD, SSD, NCC) over the disparity range. Depth is then:

$$Z = \frac{fB}{d}$$

where \(f\) is the focal length and \(B\) is the stereo baseline. Depth is inversely proportional to disparity — large disparity means the point is close.

Q12 Gamma correction equation and its purpose
$$I_{\text{out}} = c \cdot I_{\text{in}}^\gamma \quad (\text{intensities normalized to } [0,1])$$

Purpose: Gamma correction compensates for the nonlinear response of cameras and displays. Cameras typically apply \(\gamma < 1\) during capture; monitors apply \(\gamma \approx 2.2\) during display. It also allows perceptually uniform brightness/contrast adjustment since human vision is nonlinear in intensity.

Q13 Proof: histogram equalization makes the CDF linear
Proof

Define the transform \(s = T(r) = F_r(r)\), where \(F_r(r) = \int_0^r p_r(w)\,dw\) is the CDF of the input intensity \(r\).

Compute the CDF of the output \(s\):

$$F_s(s) = P(S \le s) = P(F_r(R) \le s) = P(R \le F_r^{-1}(s)) = F_r(F_r^{-1}(s)) = s$$

Since \(F_s(s) = s\) on \([0,1]\), the output CDF is linear, which corresponds to a uniform PDF — i.e., equal numbers of pixels at each grey level. Hence proved.

Q15 Proof: any LSI operator is a convolution (see Q3 Part A)

This is identical to Exercise 3 (Part A) above. For completeness:

  1. Define impulse response \(h = T\{\delta\}\).
  2. Decompose \(f = \sum_k f[k]\,\delta(\cdot-k)\).
  3. By linearity: \(T\{f\} = \sum_k f[k]\,T\{\delta(\cdot-k)\}\).
  4. By shift-invariance: \(T\{\delta(\cdot-k)\} = h(\cdot-k)\).
  5. Therefore \(T\{f\} = \sum_k f[k]\,h(\cdot-k) = f*h\). Hence proved.
Q16 Bilateral filter equation
$$I'(p) = \frac{1}{W_p}\sum_{q\in\Omega} \underbrace{\exp\!\left(-\frac{\|p-q\|^2}{2\sigma_s^2}\right)}_{\text{spatial weight}} \cdot \underbrace{\exp\!\left(-\frac{(I(p)-I(q))^2}{2\sigma_r^2}\right)}_{\text{range/intensity weight}} \cdot I(q)$$

Normalization factor:

$$W_p = \sum_{q\in\Omega} \exp\!\left(-\frac{\|p-q\|^2}{2\sigma_s^2}\right) \exp\!\left(-\frac{(I(p)-I(q))^2}{2\sigma_r^2}\right)$$

\(\sigma_s\) controls spatial extent; \(\sigma_r\) controls the intensity similarity threshold. Pixels with similar intensity but far away contribute less spatially; pixels spatially close but with very different intensity are down-weighted by the range kernel.

Q17 Is the bilateral filter a convolution?
No

A convolution requires a fixed, shift-invariant kernel. In the bilateral filter, the range weight \(\exp\!\left(-\frac{(I(p)-I(q))^2}{2\sigma_r^2}\right)\) depends on the actual pixel values at \(p\) and \(q\), so the kernel changes at every pixel location and with every image — it is data-dependent, nonlinear, and not shift-invariant. Therefore, it cannot be written as a convolution.

Q20 Shape in Hough space for a single edge pixel (circles with gradient)

For a single edge pixel \((x,y)\) with known gradient direction \(\theta\), the set of valid circle centers lies on two opposite rays (along \(\pm\theta\)) emanating from \((x,y)\). As the radius \(r\) varies, each ray traces a half-line in 2D center space \((a,b)\).

In the full 3D accumulator \((a, b, r)\), this becomes a curve (two branches) parameterized by \(r\):

$$a(r) = x \mp r\cos\theta, \quad b(r) = y \mp r\sin\theta$$

Compare to the case with unknown gradient: there, valid centers form a full circle of radius \(r\) in \((a,b)\) space for each \(r\), i.e., a cone in 3D — much more diffuse and harder to detect.

Q21 Spectral clustering for \(k > 2\) (pseudocode)
Input: Data points, number of clusters k 1. Build affinity (similarity) matrix W ∈ ℝⁿˣⁿ e.g. W(i,j) = exp(−||xᵢ−xⱼ||² / 2σ²) 2. Compute degree matrix D: D(i,i) = Σⱼ W(i,j) 3. Compute normalized Laplacian: L_sym = I − D^(−½) W D^(−½) 4. Compute the k smallest eigenvectors u₁,…,uₖ of L_sym → form embedding matrix U ∈ ℝⁿˣᵏ (columns = eigenvectors) 5. Row-normalize U: ũᵢ = uᵢ / ||uᵢ|| for each row i 6. Run k-means on rows of Ũ → cluster labels 1…k 7. Assign original points by their k-means label
Why use eigenvectors? The Laplacian eigenvectors capture global graph structure (cluster connectivity), enabling k-means to find clusters that are not linearly separable in the original feature space.
